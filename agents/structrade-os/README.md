# StrucTrade OS

StrucTrade OS 是一个面向 AI Agent 的个人投资操作系统规格。它的目标不是提供单一分析能力，而是把一个 Agent 展开为完整的美股投资辅助系统，覆盖信息摄入、研究提炼、风控约束、触发提醒、交易计划与复盘归因。

## 如何使用

最简单的使用方式，是把下面这段内容直接复制给 Agent。

可直接复制给 Agent：

```text
请读取并执行这个文档：
https://raw.githubusercontent.com/zxsun2022/UsingAI/main/agents/structrade-os/StrucTrade_OS.md

要求：
1. 严格按照文档中的 Phase 顺序逐步执行。
2. 每完成一个 Phase，先向我汇报结果。
3. 在我确认之前，不要进入下一个 Phase。
```

如果目标 Agent 不支持直接读取 URL，就直接粘贴 `StrucTrade_OS.md` 全文，并附上同样的执行要求。

## 投资理念

StrucTrade OS 不是预测市场的机器，而是一套帮助个人投资者提升决策质量的操作系统。它强调的不是更频繁地下判断，而是把判断变得更结构化、更可回溯、更能控制风险。

- 先有结构，再有观点：零散信息必须先进入统一的数据对象，再进入分析和决策流程。
- 先管风险，再谈收益：任何交易想法都要先经过反方视角和纪律检查，而不是顺着情绪加码。
- 观点必须可证伪：每个 thesis 都应该说明自己为什么成立、在什么条件下失效。
- 组合要匹配认知：仓位大小、建仓节奏和持有周期，应和用户的把握度、波动承受力与时间框架一致。
- 人做最终决策：系统负责记录、提示、质疑和校验，不代替用户下单。

## 方法框架

这套系统融合的是一种偏机构化的个人投资框架，核心不是“找一个万能指标”，而是把研究、执行和复盘连成闭环。

1. Intake：把聊天、链接、截图、观察和新闻转成结构化研究材料。
2. Thesis：从原始材料提炼可辩论、可追踪、可更新的投资论点。
3. Risk：在建仓前、加仓前、减仓前进行制度化风控审查。
4. Trigger：把关键价格位、基本面变化、事件节点和时间窗口转成提醒条件。
5. Trade Plan：将观点转为具体执行方案，包括仓位、分批、止损、失效条件和预期收益风险比。
6. Attribution：事后拆解收益来自判断、时点、仓位还是运气，持续修正方法。

## 适合的投资风格

StrucTrade OS 更适合有明确主观看法、愿意持续记录与复盘、希望建立个人交易纪律的美股投资者。它尤其适合以下场景：

- 以个股和 ETF 为主，偶尔延伸到期权或加密资产
- 既做研究驱动的中线仓位，也管理短线战术操作
- 希望把“想到一个机会”升级为“形成一套可执行、可复盘的投资流程”

## 目录定位

这个目录属于 `agents/`，不是 `skills/`。

- `skills/` 适合封装单一、可复用、边界清晰的能力模块。
- `agents/structrade-os/` 适合承载完整 agent 规格、系统角色分工、数据库设计与执行流程。

## 文件说明

- `StrucTrade_OS.md`：核心规格书，供 OpenClaw 或其他 AI Agent 读取并逐阶段展开实现。

## 使用建议

1. 将 `StrucTrade_OS.md` 作为目标 Agent 的主规格输入。
2. 优先使用 raw URL，其次再考虑直接粘贴全文。
3. 如果要给不同平台使用，优先保留该目录整体结构，而不是只分发单个文件。
