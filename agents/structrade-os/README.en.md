# StrucTrade OS

Language: [中文](./README.md) | [English](./README.en.md)  
Specs: [中文 Spec v3.0](./StrucTrade_OS.md) | [English Spec v3.0](./StrucTrade_OS.en.md)

StrucTrade OS is a personal investing operating system specification for AI agents. It is not a stock-picking bot and it is not an auto-trading system. It is a discipline-first charter that connects research, trade planning, risk controls, alerts, snapshots, and review into one operating model.

## What Changed In v3.0

- The documentation now uses a **charter layer + implementation layer** split. The spec defines principles, entities, flows, and rules; it no longer embeds SQL implementation details.
- **Exit planning is now mandatory before entry**. A TradePlan cannot be activated unless stop loss, scale-out or take-profit logic, and thesis-invalidation exit conditions are all defined.
- All major v2.0 capabilities remain, but the emphasis moves from hard-coded implementation details to agent-generated implementation that still cannot bypass discipline.

## System Positioning

StrucTrade OS is not mainly about gathering more information. Its purpose is to enforce structure and discipline when investor emotions are strongest.

- Research must become structured records before it becomes a decision.
- Every thesis must be falsifiable, not just persuasive.
- Every trade must pass risk review before return expectations matter.
- The system advises; it never auto-executes.
- Review and attribution are part of the operating loop, not optional extras.

## The v3.0 Structure

v3.0 explicitly separates the system into two layers:

- **Charter layer**: shared by humans and agents, defining system intent, roles, principles, entities, decision flows, and the trading constitution
- **Implementation layer**: generated by the agent from the charter, including schema design, migrations, indexes, views, SQL, and code

That makes the spec easier to reuse across agent frameworks without binding it to one concrete implementation.

## Good Fit

StrucTrade OS is best suited for users who:

- already have views and want a more structured investing process
- are willing to keep feeding research materials, holdings, and trade plans into the system
- want hard risk controls and trading discipline rather than note-taking alone
- need Thesis, TradePlan, Snapshot, Behavior, and Attribution to live in one loop

## Directory Role

This directory belongs under `agents/`, not `skills/`.

- `skills/` are better for narrow, reusable capabilities
- `agents/structrade-os/` is the right home for a full agent charter, operating model, and execution workflow

## Distribution Notes
